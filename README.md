# Election Night — Real-Time Simulator

A desktop-first, broadcast-style web app that turns the attached spreadsheet
election model into a **live election night** you can watch unfold minute by
minute — poll closings, vote returns, margin swings, state calls, flips,
breaking-news alerts and the race to a majority — in the spirit of CNN / BBC /
Sky election coverage.

Press **Start Election Night** and watch the count come in.

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # production bundle in dist/
```

---

## 1. The spreadsheet model (source of truth)

The workbook (`election_simulator.xlsm` + `ElectionMacros.bas`) models a
**synthetic 36-state, six-party, first-past-the-post nation** with **602 seats**
(**302** for a majority). It is a bottom-up demographic forecast, not random
numbers. The pipeline, sheet by sheet:

| Sheet | Role |
|---|---|
| **Demographics** | Per-state composition across 7 groups + national group turnout & party *leans*. Demographic-implied support = Σ(share × turnout × lean), normalised. |
| **Voting History** | Last three elections per state (used recency-weighted: 0.2 / 0.3 / 0.5). |
| **Polling** | Current per-state polls. |
| **Model** | Blends **Demographics 30% / History 25% / Polling 45%** → applies **Incumbency** swing → **Candidate** effect → **Tactical** squeeze → renormalises to final per-state win shares (`BX:CC`). Also computes composition-weighted **turnout** (`AA`). |
| **Incumbency** | One party defends power; eight Yes/No factors (economy, scandal, unrest, leader charisma…) sum to a net swing added to the incumbent's share everywhere. |
| **Tactical & Candidate** | Tactical voting squeezes parties outside a state's top two toward the strongest rival of a nominated party; candidate effect nudges per-party share. |
| **Engine** | Per **seat**, draws `RAND()` and awards it to the first party whose cumulative *(share × seat-efficiency)* threshold exceeds the draw. |
| **Simulator** | Front page: votes cast, seats per party (`SUMPRODUCT` over the Engine draws), state winners, popular vote, majority/largest-party verdicts. |
| **Coalition** | Assigns parties to blocs; totals bloc seats to decide who governs. |
| **Monte Carlo** | Repeats the seat draw thousands of times (the `RunMonteCarlo` macro). |

**Seat efficiency (FPTP distortion):** each party's per-seat probability is
scaled by an efficiency factor before allocation — Conservatives /
Social-Democrats / BP First 100%, Liberals / Nationalist 22% (spread-out support
converts poorly), Independent ~4.5%. The popular vote uses *true* share, so vote
and seat totals diverge exactly as in real FPTP systems.

The key insight reproduced here: **per-state final vote shares are
deterministic; the seat allocation is probabilistic** (proportional-with-noise ×
efficiency). The workbook re-rolls on F9 — we re-roll on a **seed**.

## 2. How this app stays faithful

* The **exact** 36 states, 6 parties, populations, turnout, seats, final vote
  shares and last-election results are extracted to [`src/data/model.json`](src/data/model.json).
* The **seat draw** in [`buildModel.ts`](src/engine/buildModel.ts) is a literal
  port of the Engine sheet / `RunMonteCarlo` macro: cumulative
  `share × seatEff` thresholds, one `rand()` per seat, first-threshold-wins.
* Verified numerically: per-state seat counts always sum to the state's seat
  count; national seats sum to 602; the live count is monotonic and converges
  **exactly** to the final draw; the same seed reproduces an identical night.

Settings (national swing, polling/turnout error, third-party boost) are the
workbook's own levers — with all errors at 0 the app reproduces the
spreadsheet's deterministic shares before the seat draw.

## 3. Turning a result into a *night*

Everything is a **pure function of (seed, virtual clock)** so play / pause /
rewind / skip / replay are trivial and never produce an impossible result:

* **Final result** computed up front from the seed (seats, per-seat winners,
  per-seat synthesised votes).
* **Reporting** — each state's seats report over the night on a smooth
  ease-in-out curve, **rural/populist seats first, urban/mail seats later**, so
  early leads erode realistically and margins swing before converging.
* **Calls** — a state is projected once reporting passes a margin-based
  threshold: safe ~2%, likely ~15%, lean ~45%, competitive ~75%, toss-up ~97%.
* **Events** — poll closings, first results, milestones, calls, **flips**,
  tightening races, largest-party and majority moments, and the final
  declaration are generated by walking the timeline (data-driven, not random).

```
scripts/

```

### The geographic map

The main view is a **vector map of the 36 states** using the **exact geometry
from [`map.svg`](map.svg)**. Each state is its real shape that fills with its
leading party's colour (opacity tracks reporting confidence), with hover detail
and click-to-pin; a **🗺 Map / ▦ Grid** toggle switches to the tile view.

`scripts/build_map_geo.py` parses `map.svg` (an Illustrator export: 41 state
paths in `Layer_3`, 36 name labels in `Layer_4`) and resolves each state's name
by which label falls inside which path (point-in-polygon). The Bras-Panon
Islands are several island paths sharing one label and render in the inset box.
**Every path's `d` string is preserved verbatim — the geometry is never
modified**; the script only computes label anchor points and the coastline
outline (the union of the mainland states) for presentation, emitting
[`src/data/mapGeo.json`](src/data/mapGeo.json). Regenerate with
`npm run build:map` (needs Python + `svgpathtools` + `shapely`).

Clean separation of **UI / engine / state machine / spreadsheet data**. The
engine is framework-free and unit-testable; the store throttles snapshots to
~10 Hz while Framer Motion tweens values for smooth, high-frame-rate visuals.

**Stack:** React + TypeScript + Vite + Zustand + Framer Motion + Tailwind.

## 5. Controls

Start / Pause / Resume / Restart / Replay · skip ±15 min · jump to end ·
0.5–8× speed · scrub the timeline · mute + volume · sortable full results
table · hover any state for live detail · click to pin it. Re-open **⚙** to
change the seed and settings and run a completely different night.
